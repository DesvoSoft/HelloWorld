Guía de Git

Domina Git para enfrentar cualquier desafío en desarrollo.


Glosario


Algunos términos clave que encontrarás en esta guía:


  • Repositorio: Un directorio que contiene tu proyecto y el historial de cambios.
  • Commit: Una instantánea de los cambios realizados en el repositorio.
  • Branch: Una rama que permite trabajar de manera independiente en nuevas funcionalidades.
  • Merge: La combinación de cambios de diferentes ramas.
  • Pull Request: Una solicitud para fusionar cambios en el repositorio principal.
  • Rebase: Reorganizar commits para mantener un historial lineal.
  • Stash: Un lugar temporal para guardar cambios sin comprometerlos.


¿Qué es Git?


Git es un sistema de control de versiones distribuido que permite a los desarrolladores gestionar y rastrear cambios en el código fuente de manera eficiente. Es una herramienta fundamental en el desarrollo de software colaborativo.

Git guarda un historial completo de cambios, lo que facilita la colaboración, el seguimiento de errores y la experimentación sin riesgo de pérdida de datos.



Configuración Inicial


Antes de empezar a usar Git, debes configurarlo en tu entorno para garantizar que tus cambios estén correctamente identificados.

# Configurar tu nombre de usuario y correo
$ git config --global user.name "Tu Nombre"
$ git config --global user.email "tuemail@ejemplo.com"

# Verificar configuración
$ git config --list

La configuración global aplica para todos los proyectos en tu sistema. Puedes usar el argumento --local para configuraciones específicas de un proyecto.


Ejemplo:


Si trabajas en dos proyectos diferentes con distintos nombres o correos, configura los valores locales para cada uno:

$ git config --local user.name "Nombre Proyecto A"
$ git config --local user.email "emailA@ejemplo.com"


Comandos Básicos


Estos comandos son esenciales para empezar a trabajar con Git:


  • git init: Inicializa un nuevo repositorio en tu directorio actual.
  • git add [archivo]: Agrega archivos al área de preparación (staging).
  • git commit -m "mensaje": Guarda los cambios en el historial con un mensaje descriptivo.
  • git status: Muestra el estado del repositorio (archivos modificados, no rastreados, etc.).
  • git log: Muestra el historial de commits. Usa git log --oneline para un resumen.

Ejemplo:

# Crear un nuevo repositorio
$ mkdir mi_proyecto
$ cd mi_proyecto
$ git init

# Crear un archivo y agregarlo al repositorio
$ echo "Hola Mundo" > archivo.txt
$ git add archivo.txt
$ git commit -m "Agregar archivo inicial"


Usos Complejos


Rebase

El rebase permite reescribir el historial de commits para mantener un historial limpio y lineal.

# Rebase de tu rama con main
$ git checkout feature-branch
$ git rebase main

# Rebase interactivo para editar commits
$ git rebase -i HEAD~3

Cherry-pick

Permite aplicar un commit específico de una rama a otra.

# Aplicar un commit a tu rama actual
$ git cherry-pick ID_del_commit

Stash

Guarda temporalmente tus cambios para trabajar en otra tarea.

# Guardar cambios en el stash
$ git stash

# Recuperar cambios guardados
$ git stash pop

Squash Commits

Combina múltiples commits en uno solo para limpiar el historial.

# Combinar commits durante un rebase interactivo
$ git rebase -i HEAD~n

# Marca los commits como 'squash' o 's'
$ git commit --amend

Conflictos en Merge y Pull Requests

Resolver conflictos es una tarea común al colaborar en Git. Sigue estos pasos:

# Intentar hacer un merge
$ git merge feature-branch

# Identificar conflictos
$ git status

# Editar archivos en conflicto y resolverlos
# Luego, agregarlos nuevamente al staging
$ git add archivo_resuelto

# Finalizar el merge
$ git commit

# En un Pull Request, comenta sobre los conflictos resueltos para facilitar la revisión.


Workflows Colaborativos


Gitflow

Un flujo de trabajo basado en ramas para gestionar desarrollos complejos.


Principales ramas:


  • Main: Código listo para producción.
  • Develop: Código en desarrollo.
  • Feature: Nuevas características.
  • Release: Preparación de nuevas versiones.
  • Hotfix: Correcciones urgentes en producción.



Ejemplo:

# Crear una rama de características
$ git checkout -b feature/nueva-funcion develop

# Fusionar rama de características
$ git checkout develop
$ git merge feature/nueva-funcion

Trunk-based Development

Un enfoque que fomenta integraciones frecuentes al branch principal.


Claves:

  • Ramas cortas y de vida breve.
  • Integraciones diarias para reducir conflictos.

Ejemplo:

# Crear una rama corta
$ git checkout -b fix/bug-123

# Fusionar rápidamente después de las pruebas
$ git checkout main
$ git merge fix/bug-123


Escenarios Avanzados


Recuperación de Commits Eliminados

Usa el reflog para recuperar commits perdidos.

# Ver el historial del reflog
$ git reflog

# Restaurar un commit perdido
$ git checkout ID_del_commit

Resolución de Conflictos Complejos

Cuando múltiples archivos tienen conflictos:

# Identificar conflictos
$ git status

# Usar herramientas gráficas (como Meld)
$ git mergetool

# Resolver manualmente y confirmar
$ git add archivo_resuelto
$ git commit

Recuperación de un Branch Eliminado

Restaura un branch eliminado utilizando su commit más reciente.

# Encontrar el commit más reciente
$ git reflog

# Crear un branch desde el commit
$ git checkout -b nombre_branch ID_del_commit

Dividir Commits

Divide un commit en varios más pequeños.

# Inicia un rebase interactivo
$ git rebase -i HEAD~n

# Marca el commit como 'edit'
$ git reset HEAD~

# Crea nuevos commits con los cambios deseados
$ git add parte1
$ git commit -m "Mensaje parte 1"
$ git add parte2
$ git commit -m "Mensaje parte 2"
